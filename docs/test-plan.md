## 2. Расписание (Schedule) — валидация

| # | Сценарий | Ожидание |
|---|---|---|
| 2.1 | Сохранить корректное расписание (Пн–Пт 09:00–18:00) | Успешно |
| 2.2 | Сохранить с `startTime >= endTime` | Ошибка |
| 2.3 | Сохранить с интервалом < 15 мин (09:00–09:10) | Ошибка |
| 2.4 | Сохранить пересекающиеся интервалы (09:00–12:00 и 11:00–13:00) | `SCHEDULE_OVERLAP` |
| 2.5 | Сохранить стыкующиеся интервалы (09:00–12:00 и 12:00–15:00) | `SCHEDULE_OVERLAP` (стык = overlap) |
| 2.6 | Сохранить интервал внутри другого (09:00–18:00 и 10:00–12:00) | `SCHEDULE_OVERLAP` |
| 2.7 | Дважды указать один день недели | Ошибка |
| 2.8 | Невалидная timeZone | Ошибка |
| 2.9 | Пустой `days` (владелец не работает) | Успешно |
| 2.10 | Пустой `intervals` в конкретном дне | Успешно (день выходной) |
| 2.11 | Добавить интервал с перекрытием — UI подсветил конфликт до отправки | Визуальная подсветка + кнопка «Объединить» |
| 2.12 | Повторный PUT с теми же данными | Успешно (идемпотентность) |

## 4. Бронирование (Booking)

| # | Сценарий | Ожидание |
|---|---|---|
| 4.1 | Гость выбрал тип → увидел свободные слоты | Слоты в окне ≤ 14 дней |
| 4.2 | dateTo – dateFrom > 14 дней | Ошибка |
| 4.3 | Выбрать слот, заполнить agenda, создать бронь | Статус `requested`, слот занят |
| 4.4 | Попробовать забронировать уже занятый слот | `SLOT_ALREADY_BOOKED` |
| 4.5 | Гость смотрит свои брони (`/bookings`) | Видит только свои |
| 4.6 | Гость отменяет подтверждённую бронь | Статус → `cancelled` |
| 4.7 | Гость отменяет уже отменённую | Ошибка |
| 4.8 | Админ подтверждает `requested` → `confirmed` | Успешно |
| 4.9 | Админ отклоняет `requested` → `rejected` | Успешно, слот свободен |
| 4.10 | Админ отменяет `confirmed` → `cancelled` | Успешно |
| 4.11 | Админ пытается подтвердить уже отклонённую | Ошибка (невалидный переход статуса) |
| 4.12 | Админ фильтрует брони по статусу | Фильтр работает |
| 4.13 | Админ фильтрует брони по дате | Фильтр работает |
