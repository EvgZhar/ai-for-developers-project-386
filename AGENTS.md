## Git
- Ветки фич, слияние в main
- Осмысленные conventional commits (feat:, fix:, chore:, docs:) на русском языке


## Требования
- Полные требования в `project.md`
- Работаем в подходе Design First
- Ключевые сущности: UserProfile, AvailabilityInterval, EventType, Booking, AuthSession

## Запуск на исполнение
- После каждого выполненного задания (промпта), связанного с UI открывать приложение через PlayWright и проверять, что результат соответствует текущему заданию
  - Если надо проверить работу "хозяина" календаря, логиниться под адином (admin@test.com / admin)
  - Если проверяем часть, касающуюся "гостя", логиниться под гостем
  - Если неочевидна роль, которая пользуется - логиниться под админом
- Если результат не соответствует ожиданиям, уточнить мне что за несоответствия выявлены в виде нумерованного списка и уточнить, надо ли все эти ситуации пробовать исправлять

## E2E-тесты (Playwright)
- Тесты в `client/e2e/`, конфиг `client/playwright.config.ts`
- Запуск: `cd client && npm run test:e2e` (запускает Vite dev server + Playwright Firefox)
- Браузер: Firefox (Chromium недоступен для скачивания, 403 от CDN)
- Данные хранятся в localStorage (Prism мок-сервер не используется компонентами)
- Хелперы: `clearAll()` (очистка localStorage), `seedProfiles()`, `seedEventTypes()`, `loginAs(page, role)`, `generateId()`
- После изменения UI-кода всегда запускать `npm run test:e2e` для проверки
